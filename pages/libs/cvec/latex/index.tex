{\ttfamily \doxylink{structCVEC}{CVEC}} is a minimal and efficient C dynamic array library that offers type-\/agnostic vector behavior with optional capacity control and safe dynamic resizing. It provides an interface similar to {\ttfamily std\+::vector} in C++ but implemented in pure C with focus on simplicity, safety, and flexibility.

\hypertarget{index_autotoc_md6}{}\doxysection{\texorpdfstring{Features}{Features}}\label{index_autotoc_md6}

\begin{DoxyItemize}
\item {\bfseries{Dynamic Capacity Management}} Initialize, resize, and grow vectors automatically, or lock capacity with {\ttfamily force\+Cap} for predictable memory limits.
\item {\bfseries{Safe Element Operations}} Supports indexed push, insert, and removal without exposing raw pointer arithmetic.
\item {\bfseries{Front, Back, and Indexed Insertion}} Add elements to the end, front, or any specific position via {\ttfamily \doxylink{cvec_8h_ac1af20c5a1c528bc5651f379386cbd0c}{\+\_\+\+\_\+cvec\+\_\+push()}}, {\ttfamily \doxylink{cvec_8h_aa150e437b43f307726e4063ce76b60e8}{\+\_\+\+\_\+cvec\+\_\+push\+Front()}}, or {\ttfamily \doxylink{cvec_8h_a65f0ba8829599270b50cdd2e63b2bc7b}{\+\_\+\+\_\+cvec\+\_\+push\+Index()}}.
\item {\bfseries{Element Access and Queries}} Retrieve or validate indices safely with {\ttfamily \doxylink{cvec_8h_a12eedad4150eeda5211fccc3817093ca}{cvec\+\_\+get()}}, {\ttfamily \doxylink{cvec_8h_aa6cb8d7b5f1186daf4b4a8def3dfbe06}{cvec\+\_\+at()}}, and {\ttfamily \doxylink{cvec_8h_ade25c4f7888a5555d6a78d1eabd38385}{cvec\+\_\+at\+Cap()}}.
\item {\bfseries{Merging and Copying}} Combine vectors or duplicate existing ones using {\ttfamily \doxylink{cvec_8h_a53ee0e61343e971ea758a475821e2b6a}{cvec\+\_\+merge()}} and {\ttfamily \doxylink{cvec_8h_a7a249c4709f3606bf78f6068a635972d}{cvec\+\_\+init\+Copy()}}.
\item {\bfseries{Deletion Utilities}} Delete by index, by element, or remove multiple matching elements through {\ttfamily \doxylink{cvec_8h_a1f946d01c379b8973d0bf4c41e01826d}{\+\_\+\+\_\+cvec\+\_\+del\+Index()}}, {\ttfamily \doxylink{cvec_8h_a97ae4722dc7a75afe22b665abb6f1f52}{\+\_\+\+\_\+cvec\+\_\+del()}}, and {\ttfamily \doxylink{cvec_8h_a10df95775694048fb7755e3ebf2be8e2}{cvec\+\_\+remove()}}.
\item {\bfseries{Reversal and Shrinking}} Reverse vector order or reduce capacity using {\ttfamily \doxylink{cvec_8h_a0bc4975070bfe7dd941c1a383e9624f7}{cvec\+\_\+reverse()}} and {\ttfamily \doxylink{cvec_8h_ac5f029841417974e9b6f1238bd2b0dd0}{cvec\+\_\+shrink()}}.
\item {\bfseries{Optional Dynamic Capacity}} When {\ttfamily dynamic\+Cap} is disabled, all operations must fit within the current capacity. This is useful for embedded or low-\/level systems.
\end{DoxyItemize}

\hypertarget{index_autotoc_md14}{}\doxysection{\texorpdfstring{Notes on Capacity}{Notes on Capacity}}\label{index_autotoc_md14}

\begin{DoxyItemize}
\item If {\ttfamily force\+Cap} is set to {\ttfamily true}, exceeding the capacity will {\bfseries{not}} reallocate memory. The function instead returns {\ttfamily CVEC\+\_\+\+FORCECAP\+\_\+\+FAIL}.
\item If {\ttfamily dynamic\+Cap} is {\ttfamily false}, the capacity remains fixed even if {\ttfamily force\+Cap} is disabled.
\item Default allocation growth is defined by {\ttfamily \+\_\+\+\_\+\+CVEC\+\_\+\+CAP\+\_\+\+ADDITION}.
\end{DoxyItemize}

\hypertarget{index_autotoc_md20}{}\doxysection{\texorpdfstring{Summary}{Summary}}\label{index_autotoc_md20}
{\ttfamily \doxylink{structCVEC}{CVEC}} simplifies handling generic dynamic arrays in C while remaining lightweight and dependency free. It’s ideal for embedded systems, lightweight applications, or any project needing a simple vector-\/like structure without C++ overhead.
